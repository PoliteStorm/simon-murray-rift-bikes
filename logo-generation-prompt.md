## Design Guidelines for All Assets

### Visual Consistency
1. **Icon Treatment**: The volcanic rift icon should be consistent across all assets (three horizontal gold lines with angular cracks)
2. **Color Usage**: Primary colors are emerald green and gold - maintain this palette
3. **Typography**: Use fonts from the recommended array, maintain consistency within asset families
4. **Spacing**: Maintain proper spacing between icon and text (minimum 20px for horizontal layouts)

### Legibility Requirements
1. **Minimum Size**: All text should be readable at the asset's intended minimum display size
2. **Contrast**: Maintain WCAG AA contrast ratios (4.5:1 for normal text, 3:1 for large text)
3. **Backgrounds**: Provide versions for both light and dark backgrounds when applicable
4. **Scalability**: All logos should be vector-based (SVG) for infinite scalability

### Technical Requirements
1. **Formats**: 
   - Primary: SVG (vector, scalable)
   - Raster fallbacks: PNG (transparent), JPG (when transparency not needed)
   - Print: PDF, EPS (vector formats)
2. **Resolution**: 
   - Web: 72-96 DPI
   - Print: 300 DPI minimum
   - Retina displays: 2x resolution versions
3. **File Naming**: 
   - Format: `rift-[asset-type]-[variant].svg`
   - Examples: `rift-logo-horizontal.svg`, `rift-twitter-banner.svg`, `rift-pfp-standard.svg`

### Brand Application Rules
1. **Do's**:
   - Use brand colors consistently
   - Maintain icon integrity
   - Ensure legibility at all sizes
   - Provide transparent backgrounds for overlays
   - Create high-resolution versions

2. **Don'ts**:
   - Don't alter the icon design
   - Don't use colors outside the brand palette
   - Don't stretch or distort logos
   - Don't place logos on busy backgrounds without proper contrast
   - Don't use low-resolution versions

## Generation Instructions for AI

When generating these assets, please:

1. **Start with the Brand Identity**: Always reference the brand colors, icon, and personality
2. **Prioritize Legibility**: Test fonts at small sizes, ensure readability
3. **Create Variations**: Generate multiple versions (light background, dark background, transparent)
4. **Maintain Consistency**: Keep the icon and color palette consistent across all assets
5. **Optimize for Use Case**: Each asset should be optimized for its specific platform/use
6. **Provide Multiple Formats**: Generate SVG (primary) and PNG (fallback) versions
7. **Include Safe Zones**: For banners, indicate safe zones where important content should be
8. **Test Scalability**: Ensure logos work at both large and small sizes
9. **Consider Context**: Think about where each asset will be used and design accordingly
10. **Document Specifications**: Include dimensions, use cases, and technical notes for each asset
